# assertNotEquals()로 동등성 비교하기
두 값이 같지 않음을 검증하는 `assertNotEquals()` 메서드는 다양한 테스트 프레임워크에 존재한다.
이 메서드의 문제는 '같지 않다'의 정의가 모호하다는 것이다.

```java
assertNotEquals(obj1, obj2);
```
여기서 `obj1`과 `obj2`가 같지 않다는 말은 정확히 어떤 의미일까?
다음과 같이 4가지 가능성이 있다.

* obj1.equals(obj2) => false를 반환한다.
* obj2.equals(obj1) => false를 반환한다.
* obj1.equals(obj2)와 obj2.equals(obj1) 모두 false 반환 
* obj1.equals(obj2)또는 obj2.equals(obj1) 중 적어도 하나가 false 반환 

잘 구현된 `equals()` 메서드는 대칭성을 띠어야 한다.
아쉽지만 유명 테스트 프레임워크 중 일부는 `assertNotEquals()`가 어떻게 작동하는지 적황히 명시하지 않는다.
게다가 그 동작 결과가 가끔 놀랍기까지 하다.
`TestNG` 프레임워크 `7.4.0에서 `assertNotEquals()`는 `assertEquals()`와 정확히 반대로 작동한다.

`assertEquals()`는 매우 엄격하게 구현된 메서드다.
obj1.equals(obj2)와 obj2.equals(obj1)를 동시에 확인하며,
obj1와 obj2 중 하나만 null 일 때는 `equals()`를 호출하지 않고 즉시 실패한다.

이러한 엄격한 요건은 자연히 `assertNotEquals()`의 성공을 느슨하게 만든다.
`assertNotEquals()`는 obj1.equals(obj2) 또는 obj2.equals(obj1) 중 하나라도 false를 반환하거나,
두 객체중 하나만 null 일때 (equals() 호출 없이) 성공한다.


> 실수 방지 가이드
* `equals()` 메서드가 대칭을 띠고 있다는 확신이 없는 한 `assertNotEquals()`는 사용하지 않는다.
`equals()`가 false를 반환하는지 확인하려면 명확하게 assertFalse(obj1.equals(obj2))를 호출하라.
* 7장에서 언급했듯이, `equals()` 메서드를 테스트할 때는 `EqualsVerifier` 등의 전용 라이브러리를 사용하라.
