# 082.md
`StringBuilder`는 생성 시 내용을 지정하거나 처음에는 비워두고 나중에 지정할 수 있다.
다음 두 코드는 동일하다.
```java
StringBUilder sb = new StringBuilder("Hello");
```
```java
StringBuilder sb = new StringBuilder();
sb.append("Hello");
```
사실 두 코드가 완전히 동일하지는 않다.
생성자 파라미터는 최초의 내부 버퍼 크기에 영향을 미치므로 성능 면에서 약간 차이가 있다.
작동 결과가 동일하기에 사람들은 `append()` 호출을 생성자로 합쳐져도 괜찮다고 여긴다.
덤으로 키보드 타이핑 횟수도 줄일 수 있다.
그러나 다음과 같이 `append()`에 `char`을 입력하면 상황이 달라진다.

```java
StringBuilder sb = new StringBuilder('[');
sb.append(value);
sb.append(']');
```

* 프로그래머가 의도한 표현
```java
new StringBuilder('[');
```
* 컴파일러가 이해한 표현
```java
new StringBuilder(91);
```

> 실수 방지 가이드 
* 문자열 리터럴을 사용해야 할 곳에서 문자 리터럴을 사용하지 않는다.
이번 절의 예시는 '`[`' 대신 "`[`"를 사용했어야 했다.
문자열 리터럴은 어떠한 일이 있어도 묵시적 변환을 거쳐 숫자가 되지 않으므로 이러한 문제가 발생할 일이 없다.
* UI 메시지를 유닛 테스트로 검사하면 이러한 문제를 신속하게 감지할 수 있다.
